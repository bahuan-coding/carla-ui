# Cliente 360° — Componentes chave

## Header / Identidade
- Avatar + nome + ID curto; badge de segmento (PF/PME); RM com avatar mini.
- Risco: semáforo (low/mid/high) + tooltip com drivers (ex.: atraso 15d, limite >80%).
- Status KYC: pill com contagem de pendências; CTA “Completar” secundário.

## KPIs resumidos
- Cards compactos (120–160 px) com métrica principal e variação: Patrimônio, Saldo, Renda estimada, Limite usado, Inadimplência.
- Usar números tabulares, sufixos (k/M), mini-sparklines discretas.

## Portfólio de produtos (tabela)
- Colunas: Produto, Limite/Valor, Utilizado, Taxa, Vencimento, Status.
- Sorting por valor/risco, sticky header, zebra suave, ícones de canal.
- Inline badge de risco do produto; botões ghost para ações rápidas (ajustar limite, suspender).

## Exposições e concentrações
- Barras horizontais por bucket (crédito, investimentos, seguros); alertas por threshold >80%.
- Mostrar “top 3 contrapartes” e “top 3 produtos” para explicar risco.

## Timeline multicanal
- Evento com ícone (email/voz/chat/app), carimbo de tempo relativo e absoluto.
- Agrupar por dia; filtros por canal e tipo (ticket, proposta, cobrança, fraude).
- CTA inline: responder, abrir task, marcar follow-up; tooltips de prévia.

## Tasks e approvals
- Checklist com estados (todo, doing, blocked, done) + SLA badge.
- Assignee com avatar; botão “assign to me”; datas com semáforo de atraso.

## Documentos e KYC
- Lista de documentos pendentes com status (faltando, revisão, aprovado).
- Upload dropzone, verificador de validade, botão “solicitar cliente” pré-formatado.

## Next Best Action (Nudge)
- Card com título, racional (por que), ganho esperado, risco esperado, confiança.
- CTA primário único; secundário “ver simulação” ghost; link para log/explicabilidade.

## Notas rápidas
- Campo curto com autosave, tags, e pinning; ordenar por atualizado recentemente.
